On this page

0. 范围与口径 ​

  • 视角：前端工程师视角 + 偏应用层的理解；少即是多
  • 自我定位：从 2023 年起陆续学 AI，2024 年开始在电商里下场折腾；但从完整 AI 体系看，我依旧是“勺子”水平（连半桶水都算不上）
  • 诚实边界：不足以碰瓷任何专业人士；这系列只写我能讲清楚、能复盘、能落地的部分
  • 心态：AI 大航海，不在船上，就在水里；我会持续向这边靠拢
  • 写作目标：用工程人的口径讲 AI——让概念变成可用直觉，让直觉落到可做事情

1. 第一章：机器学习 + 深度学习的浅尝即止（科普向） ​

  • 写作目标：用尽量少的数学，解释“AI 到底在干什么”和“为什么能学”
  • 传统路线：特征工程、线性模型、树模型、SVM 的直觉与适用边界
  • 深度学习一句话：用可微分的层堆起来，靠反向传播把误差往回传
  • 必须认识的一些名词（只讲直觉）：
    • 损失函数：你到底在优化什么
    • 梯度下降：怎么往“更好”的方向走
    • 反向传播：怎么把梯度高效算出来
    • 激活函数：为什么“非线性”是必要的
  • 工程直觉：数据、算力、评估口径决定了你能走多远

2. 第二章：扩散模型（DDPM → LDMs → DDIM → DiT）的直觉与类比 ​

  • 写作目标：把“加噪/去噪”的故事讲清楚，能让人一眼看懂每一步在做什么
  • DDPM：从干净图到纯噪声，再从噪声一步步还原
  • LDMs：为什么在 latent 里去噪更划算（省算力/省显存/更可控）
  • DDIM：为什么能更快（以及它牺牲了什么）
  • DiT：为什么用 Transformer 去做去噪器会成为趋势
  • 类比要求：每个概念都配一个形象类比（“擦黑板”“修复老照片”“逆向渲染”等），并明确类比失效边界

3. 第三章：我在电商折腾的扩散模型应用（工程补充） ​

  • 写作目标：补上“从原理到落地”的那条工程链路：数据怎么来、参数怎么控、效果怎么验
  • 常见能力拼图：
    • Mask / Inpaint / Outpaint：怎么控制“改哪里、怎么改”
    • 商品详描图实验：Figma → JSON（留坑位）→ SD 填充图片与文字
    • 布局探索：做过但效果不佳，顺带解释为什么难（约束、数据分布、评估口径）
    • Nano 的降维打击：引出“电商里哪些方向不适合做”
  • 虚拟试衣：dwpose/openpose + densepose + 3D 建模，用“工程拼装”的视角讲管线
  • 前端视角的关键：如何把“可调参数”做成可用产品能力，把输入约束讲清楚，不局限于后端
  • 结论倾向：工程要相信机器学习——先把问题变成可学习的形式，再谈模型

4. 第四章：LLM 科普向原理（够用就好） ​

  • 写作目标：讲清楚 Transformer、上下文窗口、token、概率这几个“核心开关”
  • Transformer：attention 的直觉、层堆叠、为什么适合序列
  • 编解码结构：encoder-decoder vs decoder-only 的最小区别
  • Latent/Embedding：向量空间不是玄学，是压缩后的表达
  • LLM 本质：概率模型 + 条件生成；“看起来像懂”并不等于“真的懂”
  • 风险：幻觉是默认行为，不是偶发 bug

5. 第五章：电商里的 LLM 落地（Agent / Prompt / A2UI） ​

  • 写作目标：把“调用 API”写成可控流程：输入口径、输出格式、失败处理、可观测与回归
  • Agent：怎么编排（规划/执行/复核）、什么时候该用、什么时候别用
  • Prompt：约束输出、建立结构、减少漂移；思维链/分步推理在工程里怎么用（以及怎么避免用成玄学）；必要时用自检与评估兜底
  • A2UI：对话 + LangGraph + JSON → XUI（UI 组件）→ 最终产出（更贴近 Markdown/Spec 的表达）
  • 经验判断：2025 大概率是 Agent 元年，但真正难点在工程治理而非模型本身

6. 第六章：我的 AI 定位与工程人的做法 ​

  • 观察：和非大模型方向的同学交流后更确信——AI 已经走进“各家各户”，大多数人关注的是“怎么用”
  • 工程人的觉悟：
    • 理解大概思想与边界，不需要先把最优化理论学到满级才开始干活
    • 边做边学、边做边交流，用结果和数据对齐认知
    • 找落地场景，解决“算法最后一公里”的工程问题
  • 例子线索：stable diffusion as shader（工程视角的类比与启发）
