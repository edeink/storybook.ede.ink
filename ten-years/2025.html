2025：十年再回首我的工作经历

On this page
🔖本文约 5627 字，阅读预计耗时 15 分钟。
ⓘAI 辅助写作声明：本文由博主构思逻辑，AI 辅助润色，双方共同校对。

这篇是 2025 年的碎碎念，主要是从“工作”这个视角审视自己：哪里做对了，哪里绕了弯路，哪里需要换一种活法。

在自我画像上，我一直是那种为代码着迷、能通宵达旦敲代码的同学。这辈子最幸运的事，就是把自己热爱的东西当成了工作。对代码有感情，但也正因为太投入，常常在心流里痴迷局部最优，反而在整体视野上吃过亏。所以在 2025 年，我开始更刻意地跳出舒适区（过去二十多年小镇做题家盲目相信只要努力就有收获），更执着于提升自己，引入更多团队视角，也把一部分实现交给 AI（感谢 AI 的发展）。

所以，趁着十年工作这个节点，回忆和评价一下过去的工作，也算一次工作层面的碎碎念。

让自己变优秀，和优秀的同学为伍 ​

基本经历上，大学里我折腾过学生在线的网站，写过 Android 抢课和绩点计算工具；用 Pygame 做过“鼠标坦克”，用 three.js 做过赛车游戏；也用 Java 搞过一些“乱七八糟的玩具”，比如基于 TCP/UDP 的你画我猜、俄罗斯方块，以及各种课程设计。经常在实验室通宵敲代码，不知道熬过多少回，又因酷爱零食，被人称为“食神”。在大学四年的时间里，我热衷当 APIer、过于自嗨，再加上高考 PTSD（害怕竞争），很少参加各类比赛，导致学了很多老掉牙的技术：真正面试时，反而没什么拿得出手的东西。如果能回到过去，我会建议自己多参加比赛，更早认知什么是“先进”。人类科技已经发展了很长一段时间，很多尝试都被反复验证失败或落后淘汰；如果不能站在“先进”的坐标系里，就好比在死海里和在高山上的同学比海拔——哪怕你是姚明，也会输得很彻底。

回头看，当时做过很多东西，虽然深度不行，但广度还是在的，至今有不少小游戏，也非常有趣（个人滤镜加持下）。但由于当时找工作的战略，可以说是一塌糊涂，导致最后借了学校的光，才找到第一份工作。

复盘一下，自己犯了什么错：

  1. 目标明确，但路径不清晰：和那些在考研、找工作中横跳的同学不同，我在大一时目标就很明确：进大公司，去 BAT。原因也很朴素：母亲作为教育工作者，常讲「穷养儿子富养女儿」，我也因此长期把家庭想象得“很苦”（客观说，我父母虽在体制内基层，但也已经超过不少家庭），总想着早点出来减轻负担。但问题在于，我并不知道进入大公司到底需要什么能力，于是只能相信“书本都是真本事”，在图书馆泡了很久，学了不少偏表层的东西。如果当时能在广度之外，选一个核心领域往下打透基础原理就好了：越底层越不容易变，反而能在新技术出现时举一反三。可惜那时我既缺深入的方法，也缺稳定的正反馈；纯“科研式”的摸黑探索，确实很苦。更关键的是，我把大学四年最大的失败，归结为没有早早和优秀的人形成稳定连接。且优秀的同学为伍——不只局限于同龄、同领域，往往“年级”越高越好。大学时我也曾跟着一位连年国奖的学长学习，但那时我们对“优秀”的定义都不够清楚，最后一起走了弯路。
  2. 目标明确，但路径不清晰：和那些在考研、找工作中横跳的同学不同，我在大一时目标就很明确：进大公司，去 BAT。原因也很朴素：母亲作为教育工作者，常讲「穷养儿子富养女儿」，我也因此长期把家庭想象得“很苦”（客观说，我父母虽在体制内基层，但也已经超过不少家庭），总想着早点出来减轻负担。但问题在于，我并不知道进入大公司到底需要什么能力，于是只能相信“书本都是真本事”，在图书馆泡了很久，学了不少偏表层的东西。如果当时能在广度之外，选一个核心领域往下打透基础原理就好了：越底层越不容易变，反而能在新技术出现时举一反三。可惜那时我既缺深入的方法，也缺稳定的正反馈；纯“科研式”的摸黑探索，确实很苦。更关键的是，我把大学四年最大的失败，归结为没有早早和优秀的人形成稳定连接。和优秀的同学为伍——不只局限于同龄、同领域，往往“年级”越高越好。大学时我也曾跟着一位连年国奖的学长学习，但那时我们对“优秀”的定义都不够清楚，最后一起走了弯路。
  3. 让自己变得优秀：和优秀的人为伍，是谁都懂的道理；但优秀同学的精力有限，而且很多人确实有“厌蠢症”。所以，不靠血缘、也不靠情绪维系的长期合作，核心是让自己也成为一个“值得合作的人”——更直白点，要让自己具备可被依赖的价值。曾经我很羞于“抱大腿”，总觉得有求于人不体面。但后来我慢慢接受：无论科研还是工作，每个人都在交换价值；当自己资源不足时，用时间去换更高阶同学的指点与反馈，本质上是一条可复制的成长路径。
  4. 自信，敢于表达自己：很长一段时间我因为自卑，害怕暴露无知被嘲笑，就变得沉默寡言。这在工作上不是一个好信号。企业协作不同于独立开发，你必须大量接触不同人群；信息的获取也依赖这些连接，其中既有噪音，也有关键线索。同时也要认清：谎话终究会被戳破。我特别喜欢字节的一条原则：坦诚清晰（当然必须承认，字节很多部门并不坦诚清晰，尤其某里系）。从囚徒困境的视角看，一个团体如果能普遍维持坦诚清晰，整体收益会更大；但一旦有人说谎，讲真话的人反而可能吃亏，于是大家就会被迫滑向不坦诚。但无论如何，我仍然更相信：对上级坦诚清晰、对下属坦诚清晰，是更有利的长期策略。在职场如此，在生活也如此：不要害怕失败，敢于把自己 show 出来，看似短期吃亏，但长期更容易获得收益。

说完了自己犯下的错误，也要认可自己做对的事——而且很可能会一直做下去，直到遇到下一个大坎：保持健康，拥有一个支持你的家庭；在身心稳定的基础上，勤奋、善于反思。

工作的一些经历 ​

聊聊工作上的事情吧。在一些“鄙视链叙事”里，互联网大厂会看轻用友。但我觉得用友并不差，至少在企业软件领域，它长期是财务 ERP 的头部公司。用友最大的特色之一是园区很大，我们常调侃“用友不是搞软件开发，是搞房地产的”。用友也曾在 00 年代非常风光，这点从我们预算报表部门里两位清华毕业的老员工身上也能感受到（听说以前还有清华博士，后来在下行周期离开了）。不过市场变化之后，用友的盈利能力下降；我进去的时候，用友已经连续亏损三年。

用友另一个很有特色的，是那套“低代码”体系——当时还不叫低代码，而是“单据模板”。盈利模式大概是：集团开发核心框架和核心业务，再由外包/子公司基于框架做面向客户的定制化开发。我们部门做的是多维预算报表核算，如果你接触过 sheet 这类产品，大概就是 Cube 多维模型 + 公式 + 界面交互。我当时也做过一个 Kero 的 MVVM 模型，一些抽象和后来在飞书 Base 的 model 接口有相似之处：Field、Row 之类。不过因为没做到“先进”（甚至只是“能用”），现在回看也没太多值得展开分享的。

然后经过一点波折，来到了金山。金山是我很喜欢的公司，倒不完全是“公司”本身，而是它在珠海、在海边。你能想象一个在济南、北京被雾霾折磨了六年的同学，第一次在海边生活时的那种感动吗？当时我从珠海站下高铁，打车（心疼我的 100 块）到公司附近，沿着海岸线开了四十分钟。那一刻我就决定：就金山了。

在金山两年多，我经历了 WebOffice 文字从 0.5 到 1、文档从 0 到 1。最爽的点是：这两款产品都在往上走，每天看到用户增长，成就感很真实。离开用友时我还在用 Kero（有点像 knockout）。其实那时我已经自学了 React，一直很想真正用起来，但用友节奏太慢，也不允许我切栈。所以当时我有两个诉求：第一是用 React；第二是做更复杂、不是表单堆砌的页面（用友也是表格，但偏 2B，体验没那么关键）。金山刚好能满足这两点。金山当时在将 Office 套件线上化，协作化。在这个场景下，前端复杂度真的可以做得很深。我印象最深的，是排版相关会用到区间树——算是我较早接触到的、团队自实现且复杂度很高的数据结构。后来也遇到过别的结构，比如刚去头条时做过基于贪心算法的 Myers Diff。但后面很多算法的内容，已经沉成了 lib（黑盒），和你自己要“白盒掌控”的那种感觉还是不太一样。

而后就是字节。虽然我喜欢金山，但触发我出去面试的原因有三点：第一，我心里一直有点“大厂执念”，觉得这辈子还是要去大公司看看；第二，和我同年入职、同部门的同学被裁了——我第一次意识到“原来公司真的会裁员”。对我这种父母都在体制内、习惯“一辈子一个单位”的家庭来说，裁员真的是很遥远的事情。第三，2020 年金山办公上市后，股价从 40 直飙到 500+，翻了十倍，身边有一些默默无闻的同事突然财富自由了，多少都有点羡慕嫉妒恨。三种因素叠加，我更想看看外面的世界是什么样。

因为字节当时是红极一时的公司，我原计划是上半年面试字节，下半年面试腾讯（其实这个策略现在看起来也挺搞笑的，正常就应该一起面），但因为拿到了字节的Offer，导致面试腾讯的计划搁置了。当时毕业五年，我面飞书已经到最后一面，但或许因为 HR 面发挥一般，综合被刷（口径是“大 Boss 审核不通过”，我也没再求证）。后来继续面头条，拿到了我想要的职级 2-2，以及涨薪 100%（但因为金山基数确实低，所以总包依然不算高；直到工作第四年经过多轮涨薪才勉强够到 2-2 的“及格线”）。但即使报酬很好，我仍然纠结了一周：就算涨薪 100%，我对金山、对珠海还是有感情的（仅限于 22 年及之前的金山，一千人有一千个哈姆雷特，不构成职业建议），最后还是离开了。

整体上，在字节的幸福度对我来说是快速下滑的。倒不完全是因为“卷”——我在字节也算卷王（经历了各种领导，但绩效大多是 M+）。最让我难受的是那种“找不到价值的卷”。字节很没有耐心：你进去或活水的一两个季度，会给不错的资源，但会立刻要求产出，不给 landing 的时间，强迫你在不了解上下文时就做高风险判断；然后在数据不够亮眼时，资源又会被迅速抽走。我在字节经历过很多次类似的循环，这几乎是刻在组织基因里的东西。很多人因为落差离开；没离开的，也可能逐渐被边缘化，成为背绩效的备胎。身边的人换了一轮又一轮，有些同事名字还没记牢就走了。我本身是性情中人，24、25 年又在两个部门经历了两轮裁员（虽然都“幸存”了，但一年前还高歌猛进、下一年就明日黄花，对我确实是一种折磨），再加上 2025 年年中手头的事情被 +2 砍掉，我索性进入开摆阶段，把精力转去做自己真正感兴趣的东西。

好了，牢骚先收一收，回到更“整体”的叙述。

我在头条做过基于 ProseMirror 的 PGC 创作者工具（类似秀米），后来开源成 Syllepsis（曾被多个部门使用，已是过去时）。当时头条很多方向都探索过一轮，能深挖的空间不算大，整体成效一般。另外我入职没多久，+2 就活水，触发了我的 PTSD，于是我也跟着转岗飞书。那时我还没完全理解字节的运转方式，只是觉得连大佬都走了，估计情况不乐观：不过我的直觉还是蛮准的，我转岗不到半年，+1 也转岗了，Mentor 也离开了字节。

头条那段时间，除了混了一个字节开源的 GitHub logo，还有一件挺有意思的事：我当时作为 Mentor 指导北大高校的学生，花了不少时间和北大双学位的同学交流，最后拿了个“高校合作优秀讲师”（貌似参与就有，但直到我离开飞书才发），也算一段有趣的经历。

在飞书，我们组负责一款类似 Flutter + React 的前端框架。这段时间我的技术深度提升得很快：每天都在研究框架和浏览器的各种底层原理，想不涨都难。感谢飞书 Rendering Engine 部门——飞书确实是字节里挺特别的一朵花：我们部门在没有特别亮眼的业务产出下也活了三年。

这套框架后来在 Base（多维表格）PC 端落地。我也因此在 Base 侧负责过 PC 看板的升级：在千行代码率的高压下，这条线的迭代质量做得很好；对比之前的画册，这次的交付质量提高了一个级别。同时我也承担过一部分部门分享的工作，算是强迫自己更关注技术实践和信息交流的“输入输出”。这一年我也拿到了在字节的第一个 M+——这对别人也许是信手拈来的事，但对我来说是一次很大的突破。因为我一直是个很容易自卑的同学，很长一段时间都在怀疑：2-2 这个职别对我来说是不是太高了，我确实很需要一点正反馈。

然后就是闹得沸沸扬扬的 2024 年飞书经历大裁员，主要方向是清理异地 Base + 去中台化。我刚好从深圳转到广州，又在中台部门，基本是“必死局”，整个部门被扬了。裁员那一刻你会更清楚地感受到：个人的力量确实很渺小。

之后我跟着领导活水去了电商。那时我已经利用晚上的时间陆续学了一年多 AI（基础理论 + 扩散模型），也特别想找一个落地场景，于是加入 AIGC 组。AIGC 是一个很有趣的方向：2024 年我埋头苦干，走通了 Figma → 模板 JSON → AIGC 生图的链路，完成了图文生成链路的开发。至今生成模板 1000+，调用次数破百万，在电商运营侧算一个小成就，也拿到那年的 M+。这套模式在 2025 年受到 “Nano Banana” 的降维打击。不过此时，我依然相信：就算被降维打击，如果团队能凝聚一心，仍然能产出不错的结果：因为技术本身就是不断迭代的。然而，当时的非技术挑战远超过我的想象。

简单说下组织背景：我当时在 Global Selling（TikTok 国际电商一级部门下的二级组织）。后来二级研发负责人（TD）加入字节后，陆续招聘了不少阿里 P8+ 的旧部，组织结构发生了很大变化，团队也开始进入“谁是嫡系”的游戏。这个过程中你会发现：旧部更容易掌握一手信息，从而做出不那么离谱的判断；信息不对称时，其他人只能花大量精力猜老板的意思，猜对还好，猜错就是一顿骂。字节很长一段时间讲究 Context Not Control，而电商全是 Control Not Context。更糟的是，即使方向判断错了，嫡系只要数据被修饰得不离谱，也可能就过去了；而其他同学会经历更多挑战，所以也有不少 Leader 选择了离开，包括我所在的 AIGC 组的前后端 Leader。

我在年中项目被砍时，TD 的原话大意是：“就这么点事（独立站），值得投入 40 人天？”但这个需求是 +1 和产品对齐过、TD 本身也是知道的；40 人天已经是极限压缩出来的（前端既写前端，也写微服务，辛辛苦苦做出了 MVP 版本），短短一个月风向一变，开发独立站就变成了“毫无价值、浪费人力”的事情。当然，我这种小喽啰够不上 P9+ 大佬的法眼，更像是产品、算法和工程内斗里被误伤的小角色。在这种环境下，作为一个想把技术做到位的人，想把事情做好确实很难。

简单聊聊 TD（解构高阶同学，对自己也是一种成长）。TD 技术出身，所以在技术评审上会很细，对方案细节也很敏感，这点值得学习。YC 有句话大意是：“大量了解技术方案，是一种更高维度的学习途径。”这个观点我认可。不过我不太喜欢国际电商里存在大量不接触代码的高 P 同学，真的不需要这么多的管理型人才：技术方案和论文一样，很多关键判断会因为各种有意无意的原因被省略，盲目跟随很容易产生技术误判。性格上，客观说 TD 并不是坏脾气的人。但因为他是后来者，管理方式非常粗暴：骂，各种骂。这某种程度上是一种懒政：本质是自己也没有清晰思路，给不出足够有吸引力的愿景，只能用 PUA 式的方式去“激活自驱”。我原领导不愧是从阿里出来的，一看情况不对，半年不到（正常活水要一年）就直接走特批回飞书了。

自 2025 年上半年苦苦求索无果之后（我很怕坐飞机，但为了独立站争取资源，还是一个人飞去上海和产品聊方案），再加上 Global Selling 在一群 P8+ 带领下日趋惨淡，到了“没活硬整”的地步，部门也从高峰 2xx 变成了 13x，我开始更深层地问自己：我到底追求什么？毫无疑问，十年过去，我仍然是那个会因为折腾各种稀奇古怪技术而快乐的人。2026 年的今天，我会给自己定一个更高的目标：接触更多优秀的人，用更先进的技术手段，去做让大家（不限于研发）更高效的事情。

2025 年下半年，不关注绩效，没了业务压力，我的技术能力反而迎来进一步突破：是的，技术和业务有时候不是等价的。一方面，我把知识获取从“个人学习”迁移到“团队学习”，开始建立更稳固的输入路径——不依赖企业人脉，而是抱紧大学同学里那些副教授们【狗头】。下半年我和北理工国政同学在舆情预测上，也和澳科林同学在 3D AIGC Proxy 学了不少内容（1 / 2；没有混到挂名，所以仅用于说明大概内容。高效探索居多，落地仍是个问题）。

当然，在老本行前端上，我更热烈地拥抱了 AI Coding。通过 Spec Coding，我完成了对 Faster 的复刻（自认为做得更好一些），也完成了 3D Proxy 网页的开发。非常感谢这一波 VibeCoding 浪潮：它让我从“上半年失意、怀疑自己是不是不行了”的焦虑里，重新找回了“我操，原来我还挺能打”的信心。当然，工作层面也还是有推进的：我参与到核价组，在 A2UI 上“耍”了一把，提出了「数据 + 动作」这个简单理念，并完成了 A2UI 在核价侧的落地。这个项目我依旧是通过 NoCoding 的方式（指我减少或不写代码，化身无情的 AI 奴隶主），完成了调研和实际落地的双尝试。

好了，2025 年已经过去。让我们更热烈地拥抱 2026 年 —— 未来一定会更好。
