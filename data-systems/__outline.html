提示：本大纲为 AI 生成草案，仅用于后续写作与总结时的提示与结构对齐，不代表正文已开始撰写。WIP 仅用于标记“正在撰写中”的内容。

0. 写作目标与边界 ​

  • 目标：把“多维表格（Sheet / Airtable-like）”当成一个系统来拆：模型、公式、视图、协作、性能与扩展
  • 边界：以多维表格/低代码为主线，涉及 OLAP/Cube 只作为“分析视角”的能力来源，不写成数据库教材
  • 交付：每个子主题最终都能落到可实现的模块与可验证的指标（性能/一致性/稳定性）

1. 多维表格是什么：从 Sheet 到“数据库 + 视图 + 公式 + 协作” ​

1.1 概念对齐：你以为是表格，其实是“事实表达” ​

  • Sheet：二维网格 + 公式 + 交互（更偏个人工具）
  • 多维表格：表 + 字段类型 + 视图（筛选/排序/分组）+ 权限 + 协作（更偏系统能力）
  • 关键差异：事实来源是谁、约束在哪里、变更怎么回放、冲突怎么解决

1.2 典型产品能力拆解（能力地图） ​

  • 数据模型：字段类型、校验、默认值、关联与引用
  • 视图系统：表格视图 / 看板 / 甘特 / 表单 / 图表
  • 自动化：触发器、工作流、通知与规则
  • 扩展与集成：脚本、插件、API、外部数据源

2. 核心数据模型：Records / Fields / Relations / Views ​

2.1 表、字段、记录：最小可表达集合 ​

  • 字段类型：文本、数字、日期、单选/多选、人员、附件、公式、引用、关联、查找（lookup）
  • 约束：必填、唯一、范围、格式、权限（读/写/列级/行级）
  • 关系：一对一/一对多/多对多；反向引用与级联行为

2.2 视图层：同一份数据，多种“观察方式” ​

  • 筛选/排序/分组：表达式能力、索引策略与增量更新
  • 分组聚合：count/sum/avg 只是开始，关键是口径与一致性
  • 权限与共享：视图级权限、可见字段、可导出策略

3. Cube 视角：多维模型、上钻下挖与口径治理 ​

3.1 维度与度量：把“看板”变成可解释结论 ​

  • 维度：层级（省/市/区、类目/子类）、时间（年/季/月/日）
  • 度量：可加性、半可加性、不可加性（库存类 vs 流量类）
  • 口径：过滤条件、去重策略、归因逻辑与版本化

3.2 上钻下挖：交互行为背后的计算模型 ​

  • drill down：从汇总到明细，需要可追溯的聚合链路
  • drill up：从明细到汇总，需要明确维度层级与汇总口径
  • slice/dice：切片与切块，本质是谓词组合与维度子空间

4. 公式引擎：解析、依赖图与增量计算 ​

4.1 公式是什么：不是字符串，是可执行的表达式树 ​

  • 语法：tokenize → parse → AST → evaluate
  • 函数：纯函数、volatile 函数、引用函数（跨表 lookup）
  • 错误：类型错误、除零、循环引用、空值传播与一致口径

4.2 依赖图：从“算一次”到“持续更新” ​

  • 依赖收集：单元格 → 单元格 / 字段 → 字段 / 记录 → 记录
  • 拓扑排序：循环检测、强连通分量与错误可视化
  • 增量重算：脏范围、批处理、分帧与后台计算

4.3 性能：大表与复杂公式的“预算化” ​

  • 计算预算：一次交互允许的耗时与可中断策略
  • 缓存：中间结果缓存、按视图缓存、按依赖层缓存
  • 并行：worker 化、分块计算与主线程交互优先

5. 存储与传输：编码压缩、列式组织与按需加载 ​

5.1 数据组织：不只是 JSON ​

  • 行式 vs 列式：筛选/聚合/排序的成本差异
  • 编码压缩：字典编码、RLE、delta、bit-packing 的直觉
  • 稀疏数据：稀疏矩阵、块压缩与缺失值策略

5.2 按需加载与更新：让大表“看起来不大” ​

  • 分页/分片：按行、按列、按视图窗口
  • 预取：滚动预取、交互预取与回退策略
  • 增量更新：patch、变更日志、局部重绘与局部重算

6. 协作与一致性：多端编辑、冲突与可回放 ​

6.1 协作模型：从“同步数据”到“同步意图” ​

  • 操作表达：cell edit / row insert / column schema change / view change
  • OT/CRDT：适用边界、工程复杂度与可验证性
  • 可回放：oplog、快照、对账与最小复现

6.2 权限与隔离 ​

  • 表/视图/字段/记录级权限与审计
  • 插件与脚本沙箱：能力边界与安全兜底

7. Node / Low Code：在表格之上搭“应用” ​

7.1 Schema 即产品：字段、校验、触发器与工作流 ​

  • 表单：收集数据的入口与校验系统
  • 自动化：触发条件、动作编排、失败重试与幂等
  • 集成：Webhook、API、外部数据源与同步策略

7.2 扩展体系：插件、脚本与公式的协同 ​

  • 插件：UI 扩展与数据能力扩展
  • 脚本：批处理、数据清洗与自定义逻辑
  • 版本：兼容矩阵、迁移与回滚

8. 拟发布文章方向 ​

  • 多维表格的“最小数据模型”：fields/records/relations/views 的可落地设计
  • 公式引擎：AST、依赖图与增量重算的工程实践
  • Cube 视角：上钻下挖、口径治理与可解释性
  • 大表性能：编码压缩 + 列式组织 + 按需加载 + 增量更新
  • 协作一致性：变更日志、回放与冲突处理
